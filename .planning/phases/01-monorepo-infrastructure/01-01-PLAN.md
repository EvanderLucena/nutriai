    Create a Spring Boot 3.5.x project with Java 21 per D-03 (package base: com.nutriai.api), D-04 (layered architecture), and D-10 (core starters only — NO Spring Security).

    **build.gradle** — Per D-10 and research STACK.md:
    - Plugins: java, org.springframework.boot 3.5.x, io.spring.dependency-management
    - group: com.nutriai, version: 0.1.0
    - java sourceCompatibility: 21
    - Dependencies:
      - spring-boot-starter-web
      - spring-boot-starter-data-jpa
      - spring-boot-starter-validation
      - spring-boot-starter-actuator
      - spring-boot-starter-test (testImplementation)
      - postgresql (runtimeOnly)
      - flyway-core + flyway-database-postgresql
      - lombok (compileOnly + annotationProcessor)
    - NO spring-boot-starter-security (deferred to Phase 3)
    - Use Gradle wrapper (gradlew)

    **settings.gradle** — rootProject.name = 'nutriai-api'

    **NutriAiApplication.java** — Standard @SpringBootApplication main class with @EnableJpaRepositories on com.nutriai.api.repository.

    **Layered package structure per D-04:**
    - com.nutriai.api.config — CorsConfig
    - com.nutriai.api.controller — HealthController
    - com.nutriai.api.dto — ApiResponse
    - com.nutriai.api.exception — GlobalExceptionHandler, ResourceNotFoundException
    - com.nutriai.api.model — UserRole (enum), Nutritionist (entity)
    - com.nutriai.api.repository — NutritionistRepository

    **CorsConfig.java** — Allow all origins in dev profile (application-dev.yml will have specific config). Allow GET, POST, PUT, DELETE, PATCH. Allow Authorization header. Max age 3600s.

    **HealthController.java** — Per D-09: GET /api/v1/health returns JSON with status, timestamp, version, and DB connection status. Inject NutritionistRepository to verify DB connectivity (execute a count query). Response shape:
    ```json
    { "status": "UP", "timestamp": "...", "version": "0.1.0", "db": "connected" }
    ```
    If DB connection fails, return status "DOWN" with db: "disconnected" and HTTP 503.

    **GlobalExceptionHandler.java** — Per D-08: @ControllerAdvice that handles:
    - ResourceNotFoundException → 404 with ApiResponse error shape
    - MethodArgumentNotValidException → 422 with field-level validation errors
    - Exception (catch-all) → 500 with generic error message
    All error responses use the ApiResponse record shape: { success: false, data: null, errors: [...], message: "..." }

    **ApiResponse.java** — Java record with factory methods:
    ```java
    public record ApiResponse<T>(boolean success, T data, List<String> errors, String message)
    ```
    Static factory methods: ok(data), error(message), validationError(errors).

    **UserRole.java** — Enum with values: NUTRITIONIST, ADMIN. Per D-05, minimal schema for Phase 1.

    **Nutritionist.java** — JPA entity per D-05 (placeholder, minimal schema):
    - id: UUID (@GeneratedValue)
    - email: String (@Column(unique=true, nullable=false))
    - passwordHash: String
    - name: String
    - role: UserRole (@Enumerated STRING)
    - createdAt, updatedAt: LocalDateTime (@PrePersist, @PreUpdate)
    - Use Lombok @Data, @Builder, @NoArgsConstructor, @AllArgsConstructor

    **NutritionistRepository.java** — Spring Data JPA repository extending JpaRepository<Nutritionist, UUID>.

    **application.yml** — Per D-19, sensible defaults:
    ```yaml
    spring:
      application:
        name: nutriai-api
      datasource:
        url: jdbc:postgresql://localhost:5432/nutriai
        username: nutriai
        password: changeme
      jpa:
        hibernate:
          ddl-auto: validate
        open-in-view: false
      flyway:
        enabled: true
        locations: classpath:db/migration
    server:
      port: 8080
    ```

    **application-dev.yml** — Dev profile overrides:
    ```yaml
    spring:
      datasource:
        url: jdbc:postgresql://localhost:5432/nutriai
        username: nutriai
        password: changeme
      jpa:
        hibernate:
          ddl-auto: validate
        show-sql: true
      flyway:
        clean-disabled: false
    logging:
      level:
        com.nutriai: DEBUG
    ```
    Override values come from environment variables (SPRING_DATASOURCE_URL, etc.) — these are mapped from the .env.example vars.

    **V1__create_initial_schema.sql** — Per D-05 and D-07, minimal Phase-1-only schema:
    ```sql
    CREATE TYPE user_role AS ENUM ('NUTRITIONIST', 'ADMIN');

    CREATE TABLE nutritionist (
        id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
        email VARCHAR(255) NOT NULL UNIQUE,
        password_hash VARCHAR(255) NOT NULL,
        name VARCHAR(255) NOT NULL,
        role user_role NOT NULL DEFAULT 'NUTRITIONIST',
        created_at TIMESTAMP NOT NULL DEFAULT NOW(),
        updated_at TIMESTAMP NOT NULL DEFAULT NOW()
    );

    CREATE INDEX idx_nutritionist_email ON nutritionist(email);
    ```

    **NutriAiApplicationTests.java** — Context load test that verifies the Spring Boot context starts.

    **HealthControllerTest.java** — Test the health endpoint returns correct JSON shape. Use @SpringBootTest with a mock datasource configuration for test profile (H2 in-memory).

    Generate the Gradle wrapper files (gradlew, gradlew.bat, gradle/wrapper/gradle-wrapper.properties) using gradle wrapper command or by providing them.

    **Important:** Do NOT include spring-boot-starter-security — it's deferred to Phase 3 per D-10 and the deferred ideas list. The health endpoint and all future endpoints in this phase are unauthenticated.
