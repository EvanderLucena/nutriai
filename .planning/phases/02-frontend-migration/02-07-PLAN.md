Add state: `const [deletingItem, setDeletingItem] = useState<string | null>(null);` to PlansView.

Wire confirmation into the existing `removeItem` flow:
- Where `PlanFoodRow` renders with `onRemove={() => removeItem(activeOptIdx, i)}`, change to `onRemove={() => setDeletingItem(activeOpt.items[i]?.food || 'este alimento')}` — this captures the food name for the modal.
- In the DeleteConfirmModal, `onConfirm` calls `removeItem(activeOptIdx, activeOpt.items.findIndex(it => it.food === deletingItem))` — or better, store the item index in state instead of the name to avoid ambiguity.

**Preferred approach**: Store `{ optIdx: number; itemIdx: number; name: string } | null` in state so the confirmation callback has exact indices:

```tsx
const [pendingDelete, setPendingDelete] = useState<{ optIdx: number; itemIdx: number; name: string } | null>(null);
```

Then in the PlanFoodRow render:
```tsx
onRemove={() => setPendingDelete({ optIdx: activeOptIdx, itemIdx: i, name: it.food })}
```

And the modal:
```tsx
{pendingDelete && (
  <DeleteConfirmModal
    name={pendingDelete.name}
    onClose={() => setPendingDelete(null)}
    onConfirm={() => { removeItem(pendingDelete.optIdx, pendingDelete.itemIdx); setPendingDelete(null); }}
  />
)}
```

Also add the same confirmation gate for `removeMeal`. Currently `removeMeal` is called directly via `onClick={(e) => { e.stopPropagation(); removeMeal(m.id); }}`. Add a second state:

const [pendingMealDelete, setPendingMealDelete] = useState<string | null>(null);
