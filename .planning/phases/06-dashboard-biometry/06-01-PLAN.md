  </read_first>
  <action>
    Create `V13__create_biometry_and_episode_history_tables.sql` with four tables:

    **biometry_assessment:**
    - `id` UUID PK, `episode_id` UUID FK → episode(id), `nutritionist_id` UUID FK → nutritionist(id)
    - `assessment_date` DATE NOT NULL
    - `weight` DECIMAL(6,2) NOT NULL, `body_fat_percent` DECIMAL(5,2) NOT NULL
    - `lean_mass_kg` DECIMAL(6,2), `water_percent` DECIMAL(5,2), `visceral_fat_level` INTEGER, `bmr_kcal` INTEGER
    - `device` VARCHAR(100), `notes` TEXT
    - `created_at`, `updated_at` TIMESTAMPTZ
    - Index on `(episode_id, assessment_date)` for ordered reads

    **biometry_skinfold:**
    - `id` UUID PK, `assessment_id` UUID FK → biometry_assessment(id) ON DELETE CASCADE
    - `measure_key` VARCHAR(50) NOT NULL, `value_mm` DECIMAL(5,2) NOT NULL, `sort_order` INTEGER NOT NULL
    - `created_at`, `updated_at` TIMESTAMPTZ
    - Index on `(assessment_id, sort_order)`

    **biometry_perimetry:**
    - `id` UUID PK, `assessment_id` UUID FK → biometry_assessment(id) ON DELETE CASCADE
    - `measure_key` VARCHAR(50) NOT NULL, `value_cm` DECIMAL(5,2) NOT NULL, `sort_order` INTEGER NOT NULL
    - `created_at`, `updated_at` TIMESTAMPTZ
    - Index on `(assessment_id, sort_order)`

    **episode_history_event:**
    - `id` UUID PK, `episode_id` UUID FK → episode(id), `nutritionist_id` UUID FK → nutritionist(id)
    - `event_type` VARCHAR(50) NOT NULL, `event_at` TIMESTAMPTZ NOT NULL
    - `title` VARCHAR(200) NOT NULL, `description` TEXT
    - `source_ref` VARCHAR(200), `metadata_json` JSONB
    - `created_at`, `updated_at` TIMESTAMPTZ
    - Index on `(episode_id, event_at)` for chronological reads

    Follow the SQL style of V4/V5: short header comment, explicit column types, foreign keys, and indexes inline.
  </action>
  <verify>cd backend && ./gradlew compileJava</verify>
  <acceptance_criteria>
    - migration contains `CREATE TABLE biometry_assessment`, `biometry_skinfold`, `biometry_perimetry`, and `episode_history_event`
    - `weight` and `body_fat_percent` are NOT NULL
    - `metadata_json` is declared as JSONB
    - foreign keys reference `episode(id)` and `nutritionist(id)` and `biometry_assessment(id)`
    - composite indexes support ordered reads per episode
  </acceptance_criteria>
  <done>Migration V13 provides all four tables with correct constraints, types, and indexes.</done>
