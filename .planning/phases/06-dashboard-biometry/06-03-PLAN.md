    Add `GET /api/v1/dashboard` returning one aggregate DTO with KPI values and `recentEvaluations`.

    **DashboardResponse** nested DTO:
    - `kpis`: `activePatients`, `attentionPatients` (warning status count), `criticalPatients` (danger status count), `averageAdherence` (nullable Double), `assessedInLast30Days` (int), `pendingAssessmentCount` (int — patients with active episode but no assessment in 30+ days)
    - `recentEvaluations`: list of `{ patientId, patientName, initials, status, assessmentDate, weight, bodyFatPercent }` — most recent 5 assessments for the nutritionist's patients

    **DashboardService** (nutritionist-scoped):
    - Query active patient count, status distribution, and average adherence from PatientRepository
    - Query recent assessments from BiometryAssessmentRepository
    - Compute assessedInLast30Days and pendingAssessmentCount

    **DashboardController** (`@RestController`, `@RequestMapping("/api/v1/dashboard")`, `@PreAuthorize("hasRole('NUTRITIONIST')")`):
    - Single GET endpoint returning `ApiResponse.ok(dashboardService.getDashboard(nutritionistId))`

    Add service tests (dashboard returns correct KPI counts, recent evaluations, handles empty state) and controller tests (envelope shape, nutritionist scoping).
  </action>

  </acceptance_criteria>
